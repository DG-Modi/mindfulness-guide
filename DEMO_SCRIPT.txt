[0:00 — HOOK & PROBLEM STATEMENT]
Today, we are always on our screens, checking notifications and feeling stressed. This leaves us with very little time for our mental health. Most wellness apps are just static trackers. They don't offer personalized guidance, they lack security checks for private data, and they cannot handle critical health safety events. (pause)

[0:40 — WHY AGENTS?]
This is why we need AI agents. Unlike standard apps, agents can understand context, delegate tasks, and run local tools securely. By using a multi-agent system, we can have one agent act as a wellness coach, another track screen time, and a main orchestrator run the show, keeping your data private and helpful. (pause)

[1:10 — SHOW THE BANNER & ARCHITECTURE]
(show assets/cover_page_banner.png)
I built the Mindfulness Guide to solve this. It is a secure wellness assistant.
(show assets/architecture_diagram.png)
Let's look at the architecture. When you send a message, it first goes to the Security Checkpoint. This checkpoint scrubs personal info like emails, checks for prompt injection, and filters medical emergency keywords.
If it is safe, the Orchestrator Node decides whether to delegate the task to the Wellness Coach or the Mood and Screen Time Tracker.
These sub-agents connect to a local MCP server to fetch breathing routines or log statistics.
Finally, before starting any exercise, the graph pauses at the Human Confirmation Node to ask for your permission. (pause)

[2:10 — THE BUILD & TECHNOLOGY]
This application was built using Google's Agent Development Kit framework. For tool integration, I created a local stdio-based Model Context Protocol, or MCP server. The client is tested using pytest and runs locally using the ADK Dev UI playground. (pause)
